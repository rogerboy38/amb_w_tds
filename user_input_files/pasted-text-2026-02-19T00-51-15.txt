## PART 1: TECHNICAL SPECIFICATION

### Task 4A: Add Variant Parsing Support
Update `parser.py` to parse variant specifications from requests like:
- "0227 10X organic IBC" -> family=0227, variant=10X, attribute=ORGANIC, packaging=1000L-IBC
- "0227 30X fair-trade drum" -> family=0227, variant=30X, packaging=200L-DRUM
- "Highpol 20/25 organic 25kg" -> recognize as specialty product
- "Acetypol 15/20 kosher drum" -> recognize as specialty product

Add to PRODUCT_FAMILIES in parser.py:
```python
"0227": {
    ...
    "valid_variants": ["1X", "10X", "20X", "30X"],  # concentration ratios
},
"0307": {
    ...
    "valid_variants": ["100X", "200X"],  # powder ratios
},
```

### Task 4B: Add Variant to ParsedSpec Population  
When parser detects a concentration ratio in the request text (e.g., "10:1", "30X", "200:1"), set `spec.variant` accordingly:
- "10:1" or "10X" -> variant="10X"
- "20:1" or "20X" -> variant="20X"  
- "30:1" or "30X" -> variant="30X"
- "100:1" or "100X" -> variant="100X"
- "200:1" or "200X" -> variant="200X"

### Task 4C: Update template_0227 to Support Multiple Concentrations
The current template_0227_master.json is hardcoded for 30:1. Update the meta section to list all valid concentrations and make the template parameterizable by concentration ratio.

### Task 4D: Update Engine FG Naming
Currently `_generate_fg_item` defaults cert to " Fair Trade" when spec.variant is None. This should be:
- If spec.variant is set: use it in the FG code (e.g., 0227-30X-ORGANIC-IBC)
- If spec.variant is None AND family has variants: default to the family default concentration
